# تقرير ترقية TECHNO MINDS — V61.0.2

## ما تم تنفيذه

- إصلاح تفعيل حساب المالك: لا توجد إيميلات مالك مكتوبة داخل الكود، والتفعيل يتطلب بريدًا موثقًا وCustom Claim باسم `admin`.
- فصل كود الطالب عن كود ولي الأمر للحسابات الجديدة، مع استمرار دعم الحسابات القديمة.
- منع فتح بوابة الطالب أو ولي الأمر قبل قبول الحجز.
- إضافة نموذج Curriculum هرمي متوافق مع البيانات القديمة: صف، ترم، وحدة، محاضرة وعناصر المحاضرة.
- إضافة Collections: `curriculum`, `units`, `lectures`, `lecture_materials`, `assignments_v2`, `assignment_questions`, `question_banks`, `bank_questions`, `monthly_exams`, `exam_questions_v2`, `student_progress`, `teacher_files`.
- إضافة قسم «إدارة المحتوى» للمدرس مع بحث، فلاتر، Pagination، إضافة، تعديل، نسخ، حذف بـModal، حالات مسودة/منشور/مخفي، ومواعيد فتح وإغلاق.
- إضافة استيراد متعدد الملفات مع معاينة، استخراج رقم المحاضرة من الاسم، حد 15MB، وحالة نجاح/فشل لكل ملف.
- إضافة «ملفات المدرس» بصلاحيات مدرس/Admin فقط في الواجهة وFunctions وFirestore وStorage.
- إضافة صفحة «منهجي» داخل بوابة الطالب، مع الترمين، ترتيب المحاضرات رقميًا، نسبة الإنجاز، وتبويبات المحاضرة.
- تسجيل فتح المحاضرة وآخر فتح ونسبة الإكمال داخل `student_progress`.
- منع القراءة المباشرة للمحتوى والإجابات؛ بيانات الطالب تخرج من Functions بعد التحقق من الصف.
- ملفات المحتوى تُفتح بروابط موقعة قصيرة العمر بعد التحقق من الطالب والصف.
- إنشاء خطة 12 امتحانًا شهريًا لكل صف كمسودات، كل امتحان يغطي ثلاث محاضرات، دون إدخال أسئلة وهمية.
- Migration آمن يدعم Dry Run، يمنع التكرار، قابل لإعادة التشغيل، وينشئ Backup تلقائيًا قبل التنفيذ الحقيقي.
- جعل Activity Log للقراءة فقط من العميل والكتابة من Functions لمنع تزوير السجل.
- إضافة Content Security Policy وتحسين التحقق من أرقام الهاتف.
- تحديث PWA Cache إلى V61 لمنع عرض ملفات JavaScript القديمة.

## التوافق مع النظام السابق

لم تُحذف Collections القديمة أو خصائص الحضور والدفع والمجموعات وQR والدرجات والبوابات. Migration ينسخ المحتوى إلى النظام الجديد ولا يحذف المصدر.

## النشر

من مجلد المشروع شغّل بالترتيب:

```powershell
npm test
npm run build
firebase deploy --only firestore:rules,firestore:indexes,storage
firebase deploy --only functions
vercel --prod
```

يمكن استخدام `npm run firebase:deploy:all` إذا كان Firebase Hosting هو الواجهة الأساسية.

## خطوات Firebase Console اليدوية

1. تأكد أن حساب المدير موجود في Firebase Authentication وأن بريده Verified.
2. اضبط Custom Claim `admin: true` للحساب الأساسي باستخدام Admin SDK مرة واحدة فقط عند الحاجة للـbootstrap.
3. انشر Firestore Indexes وانتظر حتى تصبح Enabled قبل تجربة صفحة المنهج.
4. تأكد من إعداد Storage Bucket الافتراضي لأن روابط الملفات المؤقتة تعتمد عليه.
5. فعّل TTL للمجموعات التي تحتوي `expiresAt` إذا لم يفعّلها النشر تلقائيًا.
6. App Check غير مفعل في هذه النسخة لأن المشروع الحالي لا يحتوي Site Key. يوصى بإنشاء reCAPTCHA Enterprise key وربطه بعد اختبار الدومين الإنتاجي؛ لا تضف مفتاحًا وهميًا.

## الترحيل

من لوحة التحكم ← إدارة المحتوى:

1. اضغط «فحص Dry Run» وراجع `scanned/create/skip`.
2. تأكد أن النسخ الاحتياطي التلقائي يعمل.
3. اضغط «تنفيذ الترحيل» مرة واحدة.
4. أعد تشغيل Dry Run؛ يجب أن تصبح العناصر الموجودة ضمن `skip` وألا تتكرر.
5. راجع المحاضرات المنقولة واجعل حالتها منشور بعد تحديد الوحدة ورقم المحاضرة الصحيحين.
