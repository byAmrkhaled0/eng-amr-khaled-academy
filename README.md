# منصة Techno Minds للبرمجة والذكاء الاصطناعي — الإصدار 60.6.2

منصة تعليمية عربية مبنية بـ HTML وCSS وJavaScript وFirebase، ومجهزة للعمل على الموبايل كتطبيق ويب قابل للتثبيت (PWA). دخول الطلاب وأولياء الأمور يتم بالأكواد، بينما حسابات فريق العمل تستخدم Firebase Authentication.

## أهم تحديثات الإصدار 60.6.2

- إصلاح عدم ظهور الواجب: Functions تراجع صف الطالب من سجل `students` الأساسي وتصلح `student_portal` القديم تلقائيًا قبل فلترة الواجبات.
- عدم عرض «0 واجب» عند فشل Function أو عدم نشرها؛ تظهر حالة خطأ واضحة بدل النتيجة المضللة.
- تبسيط صفحة الدفع إلى كارت لكل طالب وزر واحد «تم الدفع»؛ يسجل المتبقي من سعر صف الطالب ويحدّث المحصل والعدادات مباشرة.
- الاحتفاظ بدفتر المعاملات الآمن خلف الزر البسيط، مع طلب حتمي لمنع التسجيل المكرر إذا انقطع الاتصال.
- نقل الوضع الليلي من زر كبير في القائمة الجانبية إلى أيقونة مستقلة في هيرو لوحة الإدارة.
- إعادة تنظيم صفحة الاختبارات بمؤشرات وكروت، ونموذج إضافة مستقل، وأقسام قابلة للطي للمحاولات والتصحيح.
- إصلاح مسار محاضرات PDF: الواجهة تجرب Hosting API ثم Firebase Callable تلقائيًا عند غياب الـrewrite في المعاينة المحلية أو النشر القديم.
- إضافة واجبات موجهة بالصف بنوعين أساسيين: اختياري يُصحح على الخادم، وكود برمجي يكتبه الطالب داخل بوابته.
- منع القراءة العامة لـ`assignments`؛ الواجب والاختيارات يصلان فقط بعد التحقق من كود الطالب وصفه، ولا تصل الإجابة الصحيحة للمتصفح.
- سجل شهري حقيقي داخل `monthly_payments` ومعاملات مستقلة داخل `payment_transactions`؛ لا تُستبدل دفعة شهر سابق.
- دعم الدفع الكامل والجزئي، مع المبلغ المطلوب والمدفوع والمتبقي وتاريخ وطريقة الدفع والملاحظات ومن سجّل العملية.
- منع الضغط المكرر بطبقتين: رقم طلب فريد في الواجهة وسجل `_payment_dedup` مؤقت على الخادم.
- تعديل وإلغاء المعاملة للمدير فقط؛ الإلغاء لا يحذفها بل يحتفظ بها في السجل و`activityLog`.
- خزنة لحظية تعرض مقبوضات اليوم والفترة والمتوقع والمتبقي وأعداد الكامل والجزئي وغير المدفوع، للطلاب النشطين فقط.
- فلاتر العام والشهر والمسار والحالة، وتصدير CSV وExcel عند الطلب من دون تحميل مكتبة Excel مع فتح اللوحة.
- ظهور سجل المدفوعات الشهرية داخل ملف الطالب وولي الأمر، مع بقاء حقول `paid/paymentAmount` القديمة للتوافق فقط.
- ترحيل Idempotent للبيانات القديمة يسبقُه Backup سحابي ولا يحذف أي سجل قديم.
- Push للحجوزات في الخلفية عبر FCM وPush API، مع تحميل Messaging عند التفعيل فقط وService Worker بلا استيرادات خارجية.
- تعطيل التوكنات المنتهية وإلغاء توكن الجهاز عند تسجيل الخروج، مع منع القراءة المباشرة لتوكنات الإدارة.
- نشر Windows قابل للاستكمال بـ `-Resume`، وينشر كل Function منفردة ليعرض اسم الدالة التي فشلت، ولا يحذف Functions قديمة ولا يدفع GitHub تلقائيًا.
- سكربت Hosting فقط، وسكربت فحص بعد النشر، وخيار اختبار Judge0 الحقيقي للغات JavaScript وPython وC++ وJava وC#.
- تحسين صورة م. عمرو خالد بنسخة WebP أخف مع JPEG fallback، واستمرار تحميل QR وExcel عند الحاجة فقط.


```text

```

## الاختبار والبناء

```powershell
npm test
npm run build
npm run verify:dist
```

نسخة النشر تُنشأ داخل `dist/`.

## النشر على Firebase

للنشر الكامل الآمن على Windows:

```powershell
.\DEPLOY-WINDOWS.cmd
```

لاستكمال نشر توقف بسبب خطأ:

```powershell
.\DEPLOY-WINDOWS.cmd -Resume
```

لنشر الواجهة فقط من دون Functions أو Rules أو Indexes:

```powershell
.\DEPLOY-HOSTING-ONLY.cmd
```

## متطلبات مهمة

- تثبيت المنصة على الهاتف يتطلب تشغيل الموقع عبر HTTPS.
- النسخ الاحتياطي المجدول يعتمد على Cloud Scheduler، وقد يتطلب خطة Firebase تدعم Cloud Functions المجدولة.
- تفعيل المصادقة الثنائية لحسابات فريق العمل يتم من إعدادات Firebase Authentication / Identity Platform.
- تفعيل Push يتطلب إنشاء Web Push certificate من Firebase Console ووضع المفتاح العام فقط في `messagingVapidKey`.
- لا يشغّل أي سكربت `git push`. راجع `README_DEPLOY.md` قبل أي رفع يدوي إلى GitHub.
- راجع `UPGRADE_V56_AR.md` قبل النشر، خصوصًا ترتيب نشر الدوال ثم الاستضافة.
